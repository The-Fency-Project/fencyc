# Small silly contributing guide 
Current tasks could be found in TODO.md
`// TODO:` comments in code are for smaller but still important ones.
# Formatting rules
Just don't do mess. Use rustfmt if possible.
Meaningful variable names, no unrelated stuff in comments...
# Opening issue
Make sure other one with the same problem aren't still open.
Provide all useful context.
# Opening pull request
Describe your improvements, why they're important. Check formatting and don't 
forget to run tests! (`cargo test`). Make sure all of them passed.
